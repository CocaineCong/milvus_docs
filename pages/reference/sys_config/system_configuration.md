# Milvus系统配置清单

本主题介绍了Milvus系统配置的一般部分。

Milvus维护了相当多的参数来配置系统。每个配置都有一个默认值，可以直接使用。您可以灵活地修改这些参数，以便Milvus能更好地为您的应用程序服务。有关更多信息，请参见[配置Milvus](configure-docker.md)。

<div class="alert note">
在当前版本中，所有参数只有在Milvus启动时配置后才会生效。
</div>

## 部分

为了方便维护，Milvus根据其组件、依赖项和一般用途将其配置分为17个部分。

### `etcd`

etcd是支持Milvus元数据存储和访问的元数据引擎。

在本节下，您可以配置etcd端点、相关的关键前缀等。

有关本节下每个参数的详细描述，请参见[etcd相关配置](configure_etcd.md)。

### `minio`

Milvus支持MinIO和Amazon S3作为数据持久性的存储引擎，用于插入日志文件和索引文件。虽然MinIO是S3兼容性的事实标准，但您可以直接在MinIO部分下配置S3参数。

在本节下，您可以配置MinIO或S3地址、相关访问密钥等。

有关本节下每个参数的详细描述，请参见[MinIO相关配置](configure_minio.md)。

### `pulsar`

Pulsar是支持Milvus集群可靠存储和消息流的发布/订阅的底层引擎。

在本节下，您可以配置Pulsar地址、消息大小等。

有关本节下每个参数的详细描述，请参见[Pulsar相关配置](configure_pulsar.md)。

### `rocksmq`

RocksMQ是支持Milvus独立模式可靠存储和消息流的发布/订阅的底层引擎。它基于RocksDB实现。

在本节下，您可以配置消息大小、保留时间和大小等。

有关本节下每个参数的详细描述，请参见[RocksMQ相关配置](configure_rocksmq.md)。

### `nats`

NATS是一种面向消息的中间件，允许以消息的形式在应用程序和服务之间交换数据。Milvus使用NATS作为可靠存储和发布/订阅消息流的底层引擎。您可以将其作为RocksMQ的替代品。

在本节下，您可以配置NATS服务器、监控属性以及保留时间和大小等。

有关本节下每个参数的详细描述，请参见[NATS相关配置](configure_nats.md)。

### `kafka`

Apache Kafka是一个开源的分布式事件流平台，用于高性能数据管道、流分析、数据集成和关键任务应用程序。它作为RocksMQ和Pulsar的替代品，用于可靠存储和发布/订阅消息流。

有关本节下每个参数的详细描述，请参见[Kafka相关配置](configure_kafka.md)。

### `rootCoord`

根协调器（root coord）处理数据定义语言（DDL）和数据控制语言（DCL）请求，管理TSO（时间戳甲骨文），并发布时间滴答消息。

在本节下，您可以配置根协调器地址、索引构建阈值等。

有关本节下每个参数的详细描述，请参见[根协调器相关配置](configure_rootcoord.md)。

### `proxy`

代理是系统的访问层和用户的端点。它验证客户端请求并减少返回的结果。

在本节下，您可以配置代理端口、系统限制等。

有关本节下每个参数的详细描述，请参见[代理相关配置](configure_proxy.md)。

### `queryCoord`

查询协调器（query coord）管理查询节点的拓扑和负载均衡，以及从增长段到封闭段的移交操作。

在本节下，您可以配置查询协调器地址、自动移交、自动负载均衡等。

有关本节下每个参数的详细描述，请参见[查询协调器相关配置](configure_querycoord.md)。

### `queryNode`

查询节点在增量和历史数据上执行向量和标量数据的混合搜索。

在本节下，您可以配置查询节点端口、优雅时间等。

有关本节下每个参数的详细描述，请参见[查询节点相关配置](configure_querynode.md)。

### `indexNode`

索引节点为向量构建索引。

在本节下，您可以配置索引节点端口等。

有关本节下每个参数的详细描述，请参见[索引节点相关配置](configure_indexnode.md)。

### `dataCoord`

数据协调器（data coord）管理数据节点的拓扑结构，维护元数据，并触发刷新、压缩和其他后台数据操作。

在本节下，您可以配置数据协调器地址、段设置、压缩、垃圾收集等。
